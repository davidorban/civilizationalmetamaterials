# Data Dictionary

Variables collected by the provenance scaffolding system and verification timer.

| Variable | Type | Description |
|---|---|---|
| `panel_id` | string | Anonymized panel identifier (e.g., `P001`) |
| `period` | int | Study period (1 = first 4-week block … 4 = last) |
| `week` | int | ISO calendar week within the study |
| `review_id` | string | Anonymized review item identifier |
| `is_ai_generated` | bool | True if summary was generated by an AI system |
| `has_class_a` | bool | True if summary has cryptographic provenance (Class A) |
| `has_class_b` | bool | True if summary has institutional provenance (Class B) |
| `has_class_c` | bool | True if summary has context binding (Class C) |
| `provenance_complete` | bool | `has_class_a AND has_class_b AND has_class_c` |
| `was_verified` | bool | True if a human reviewed the summary before decision |
| `verification_time_s` | float | Time (seconds) human spent on verification; NaN if not verified |
| `decision` | enum | `adopt`, `reject`, `defer` |
| `is_calibration` | bool | True if this is a synthetic calibration application |
| `cohort` | char | `A`, `B`, or `C` — randomization cohort |
| `treatment` | bool | True if panel was in intervention at time of review |
| `r_eff_hat` | float | Estimated R_eff for this panel-period (computed post-hoc) |
| `crossed` | bool | True if R_eff_hat < 1 for ≥ 2/4 weekly estimates in this period |

## Derived variables

| Variable | Formula |
|---|---|
| `rho_hat` | `mean(provenance_complete)` per panel-period |
| `tau_hat` | `mean(was_verified)` per panel-period |
| `r_eff_hat` | `r_eff(beta_hat, rho_hat, tau_hat, gamma_hat)` from `cm.constitutive` |

## Notes

- All `review_id` values are SHA-256 hashes of the original identifier; the
  mapping is stored offline by the panel chair and not shared with the research team.
- Calibration applications (`is_calibration = True`) are excluded from all
  primary and secondary analyses; their data are used only for calibration of
  the measurement instruments.
- Missing `verification_time_s` for verified items (e.g., timer not started)
  is imputed using the panel's median verification time.
